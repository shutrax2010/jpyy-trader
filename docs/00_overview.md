## 1. アプリの概要

### JPYY TRADER とは

**JPYY TRADER** は、ブロックチェーン（Polygon）上のトークンを AI が自律的に売買するデモアプリです。

Claude AI がリアルタイムで市場を分析し、「買う・売る・様子見」を判断してトランザクションを自動送信します。その動作をブラウザで観覧することができます。

```
                  ┌─────────────────────┐
                  │   Claude AI         │
                  │   （売買判断）        │
                  └──────────┬──────────┘
                             │ BUY / SELL / HOLD
                  ┌──────────▼──────────┐
                  │   エージェント       │
                  │   ウォレット         │
                  └──────────┬──────────┘
                             │ トランザクション送信
                  ┌──────────▼──────────┐
                  │   Polygon Amoy      │
                  │   Testnet           │
                  │   JPYY / YTT / AMM  │
                  └─────────────────────┘
                             │ リアルタイム配信
                  ┌──────────▼──────────┐
                  │   ブラウザ画面       │
                  │   （閲覧・管理）     │
                  └─────────────────────┘
```

### デモとしての位置づけ

- Polygon **Amoy Testnet** を使用するため、実際のお金は一切かかりません
- テスト用トークン（JPYY・YTT）とテスト用 POL（ガス代）を使います
- AI が実際にブロックチェーンにトランザクションを送信する「本物の動作」を観覧できます

## 2. 基本的な機能

### 機能一覧

| 機能 | 説明 | 操作者 |
|------|------|-------|
| AI自律売買 | Claude AIがJPYY/YTT の売買を自動判断・実行 | AI自動 |
| リアルタイム価格表示 | YTT価格の推移チャートをリアルタイムで表示 | 全員が閲覧 |
| AI判断ログ | AIの売買判断と理由をリアルタイムで表示 | 全員が閲覧 |
| TX履歴 | トランザクションの一覧・状態をリアルタイムで表示 | 全員が閲覧 |
| AIモード切替 | 積極モード・慎重モードを切り替え | 担当者が設定 |
| JPYY発行・配布 | デモ用JYPYトークンを発行してウォレットへ配布 | 管理者のみ |
| AMM流動性管理 | AMMプールへの流動性追加・価格変更 | 管理者のみ |

### 閲覧モードについて

トレード画面（`/`）は**エージェントウォレットをMetaMaskで接続して使用します**。接続したウォレットがエージェントとして動作し、JPYY・YTT の売買を行います。AIエージェントの起動前に必ずウォレット接続が必要です。

## 3. トークンの説明

このアプリでは2種類のトークンを使います。

### JPYY（ジェーピーワイワイ）

```
1 JPYY = 1円
```

- 日本円に連動した**基本通貨トークン**です
- AIエージェントはこれを使って YTT を購入します
- 管理者が任意の量を発行（mint）できます
- デモ参加者のウォレットに配布して使用します

### YTT（ワイティーティー）

```
価格は変動します（AMMプールの比率で決まる）
```

- AIが売買する**変動トークン**です
- 価格は `JPYY残高 ÷ YTT残高` で自動計算されます
- AIがたくさん買うと価格が上がり、売ると価格が下がります
- 価格の変動がAIの次の判断に影響します

### AMMプール（自動マーケットメーカー）

JPYY と YTT の交換レートを自動で決定する仕組みです。

```
価格計算式（定数積方式）

JPYY残高 × YTT残高 = 一定（k）

例:
  JPYY = 10,000 / YTT = 100
  → YTT価格 = 10,000 ÷ 100 = ¥100

AIが ¥1,000 分のJPYYで YTT を買った後:
  JPYY = 11,000 / YTT ≈ 90.9
  → YTT価格 = 11,000 ÷ 90.9 ≈ ¥121（価格が上昇）
```

## 4. ウォレットアカウントの説明

このアプリで使うウォレットは**2種類だけ**です。

```
┌─────────────────────────────────────────────────┐
│  管理者ウォレット                                 │
│  （.env で秘密鍵を管理・バックエンドが署名）       │
│                                                  │
│  ・コントラクトのデプロイ                         │
│  ・JPYYの発行（mint）                            │
│  ・エージェントウォレットへの JPYY 配布           │
│  ・AMMへの流動性追加・価格変更                   │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│  エージェントウォレット                           │
│  （起動時にMetaMask接続・以降は自動）             │
│                                                  │
│  ・Claude AIの判断を受けてスワップを実行          │
│  ・JPYY・YTT の残高を保有                       │
│  ・トレード画面の「HOLDINGS」に残高が表示される   │
└─────────────────────────────────────────────────┘
```

### なぜ2種類だけ？

旧設計では「オーナー・管理者・ユーザー・エージェント」と4つのウォレットがありましたが、**デモ用途に特化してシンプル化**しました。

- オーナー + 管理者 → **管理者ウォレットに統合**
- ユーザー + エージェント → **エージェントウォレットに統合**（AIが売買し、その残高がデモの「保有状況」になる）

### 必要な残高

| ウォレット | 必要なトークン | 用途 |
|-----------|-------------|------|
| 管理者 | POL（少量） | ガス代（デプロイ・管理操作） |
| 管理者 | JPYY（発行分） | エージェントへの配布用 |
| エージェント | POL（少量） | ガス代（スワップのたびに消費） |
| エージェント | JPYY（¥50,000相当） | BUY時の支払い |
| エージェント | YTT | SELL時の支払い（売買で自動増減） |

### ウォレットの接続方法

| ウォレット | 接続方法 | 用途 |
|-----------|---------|------|
| 管理者 | `.env`（秘密鍵を環境変数に設定） | バックエンドが自動操作・署名 |
| エージェント | **MetaMask接続必須**（トレード画面起動時に毎回） | フロントエンドで接続。管理ウォレットと同じアドレスは接続不可。 |

## 5. AIエージェントの説明

### 何をするのか

Claude AI がブロックチェーン上の価格データを見て、トークンの売買を自動で判断・実行します。

```
30〜300秒ごとに以下を繰り返す

  1. AMMプールから YTT 価格を取得
       ↓
  2. 直近の価格推移・変化率・トレンドを計算
       ↓
  3. Claude API に送信
     「価格が ¥95 で -5.2% 下落中。買うべきか？」
       ↓
  4. Claude が判断を返す
     { "action": "BUY", "reason": "割安水準、反転期待", "confidence": 78 }
       ↓
  5. BUY/SELL ならスワップトランザクションを送信
       ↓
  6. 結果をブラウザにリアルタイム配信
```

### AIへの入力情報

Claude API には毎回以下の情報を渡します。

```
現在のプール状態:
  JPYY残高: ¥10,200
  YTT残高:  100.00 YTT
  YTT現在価格: ¥102.00

直近10件の価格推移:
  ¥98 → ¥99 → ¥100 → ¥101 → ¥102（上昇傾向）

変化率:
  前回比: +1.0%
  10件平均比: +2.0%
  トレンド: 上昇

エージェントの残高:
  JPYY: ¥12,400
  YTT:  120 YTT
  設定取引量: ¥1,000
```

### AIの判断結果

```json
{
  "action": "BUY",
  "reason": "価格が上昇傾向。YTTを追加購入して利益を狙う。",
  "confidence": 75
}
```

- `action`: BUY（購入）/ SELL（売却）/ HOLD（様子見）
- `reason`: 判断した理由（日本語で表示）
- `confidence`: 確信度（0〜100%）

### AIモード

売買の積極性を2段階で切り替えられます。

```
積極モード 🔥
  └ 価格変化率が ±3% を超えたら売買
  └ HOLDは最小限、積極的に行動
  └ 取引量: 設定量の100%を使用

慎重モード 🛡
  └ 価格変化率が ±8% を超え、かつ明確なトレンドがある場合のみ売買
  └ 不確実なときはHOLD優先
  └ 取引量: 設定量の50%を使用
```

### 取引量の設定

1回の売買で使う JPYY の量をテキストボックスで設定します。

- 最小: ¥100
- 最大: ¥50,000
- エージェントの JPYY 残高を超える設定はエラーになります

エージェントウォレットを MetaMask で接続してから使います。接続したウォレットがエージェントとして売買を行います。

```
┌────────────────── HEADER ─────────────────────────────┐
│  JPYY●TRADER   AMOY TESTNET   ● RUNNING   積極モード   │
│                                              管理 →    │
├──────────────┬────────────────────────────────────────┤
│              │                                        │
│  AIエージェ  │  ┌──── YTT価格チャート ──────────────┐ │
│  ント制御    │  │  ¥120.00  ↑+2.04%               │ │
│              │  │  ～～～～/\/\/\/～               │ │
│  AI判断:     │  └──────────────────────────────────┘ │
│  ┌────────┐ │                                        │
│  │BUY YTT │ │  ┌── AMMプール ──┐ ┌── AI判断 ──────┐ │
│  └────────┘ │  │ JPYY ¥10,000  │ │  BUY YTT       │ │
│  "割安水準…" │  │ YTT  100.00   │ │  "割安水準…"   │ │
│              │  │ ¥100.00/YTT   │ │  Confidence 78%│ │
│  モード:     │  └───────────────┘ └────────────────┘ │

│              │  ┌── TX履歴 ──────────────────────────┐│
│  取引量:     │  │ 12:34  BUY  JPYY→YTT  ✓Done      ││
│  ¥[1,000  ] │  │ 12:33  BUY  JPYY→YTT  ⟳Pending   ││

│  [STOP AGENT]│  └────────────────────────────────────┘│

├──────────────┤                                        │
│  HOLDINGS    │                                        │
│  アドレス    │                                        │
│  0x1A2B...   │                                        │
│  POL  0.98   │                                        │
│  JPYY ¥12,400│                                        │
│  YTT  120.00 │                                        │
│  合計≈¥26,800│                                        │
└──────────────┴────────────────────────────────────────┘
```

#### 各パネルの説明

**AIエージェント制御パネル（左上）**

- 現在のAIの判断（BUY/SELL/HOLD）と理由が表示されます
- AIモードの切り替えができます
- 取引量をテキストで入力できます
- START/STOP でエージェントを起動・停止できます
- 次回実行までのカウントダウンが表示されます

**HOLDINGS パネル（左下）**

- エージェントウォレットの残高をリアルタイムで表示します
- POL（ガス代）・JPYY・YTT の残高と合計資産（¥換算）が見えます
- MetaMask 接続後に自動的に接続アドレスの残高を表示します

**YTT価格チャート（右上）**

- YTT の価格推移を折れ線グラフでリアルタイム表示します
- BUY（緑のドット）/ SELL（赤のドット）が発生したポイントをグラフ上に表示します

**AMMプールパネル（右中左）**

- プール内の JPYY / YTT 残高を表示します
- YTT 現在価格と、入力量に対する予想出力を表示します

**AI判断パネル（右中右）**

- Claude API が返した最新の判断を表示します
- 判断理由・確信度・モデル名・判断時刻が確認できます

**TX履歴パネル（右下）**

- 過去のトランザクションを一覧表示します
- Pending（処理中）→ Done（確定）の状態変化をリアルタイムで追えます
- TX ハッシュをクリックすると Polygonscan で詳細を確認できます

## 7. 管理画面の使い方

管理画面（`/admin`）はアクセスにパスワード等は不要ですが、操作はバックエンドの `.env` に設定された管理者ウォレットが署名します。誤操作防止のため確認ダイアログを必ず表示します。

### JPYY 管理

**JPYY の発行（mint）**

デモに使う JPYY を新規発行します。

```
1. 発行量を入力（例: ¥50,000）
2. 発行先を選択（エージェントウォレット or カスタムアドレス）
3. [発行する] ボタンを押す
4. バックエンドが管理者ウォレット（.env）で署名して送信
5. TX確定後、残高に反映される
```

**JPYY の配布（transfer）**

管理者の残高から他のアドレスへ JPYY を送ります。

```
1. 配布先アドレスを入力（複数可）
2. 1アドレスあたりの配布量を入力
3. 合計量を確認して [配布する] を押す
```

### AMM 管理

**流動性の追加（addLiquidity）**

プールの JPYY・YTT 残高を増やします。価格比率を維持したまま増やすのが基本です。

```
現在の比率 JPYY:YTT = 100:1 の場合
  JPYY を +5,000 追加するなら YTT も +50 追加する
  → 価格は変わらず、プールの厚みだけ増える
```

**価格のリバランス（setReserves）**

YTT の価格を直接設定します。デモ用の特別機能です。

```
1. 目標価格を入力（例: ¥150）
2. プレビューで変更内容を確認
3. AIへの影響シミュレーション（どのモードでトリガーされるか）を確認
4. [価格を設定する] を押す
```

> ⚠ setReserves は k 定数が変わるデモ専用機能です。本番環境では使用しません。

### 操作ログ

全管理操作の履歴を確認できます。発行量・配布先・TX ハッシュが記録されます。

## 8. デモの準備と開始手順

### 事前準備（初回のみ）

```
Step 1  2つのウォレットを作成
        ├── 管理者ウォレット（MetaMask）
        └── エージェントウォレット（MetaMask）

Step 2  Faucet でテスト POL を補給
        https://faucet.polygon.technology/
        └── 管理者・エージェント両ウォレットに POL を取得

Step 3  コントラクトをデプロイ
        npx hardhat run scripts/deploy.ts --network amoy
        └── JPYY / YTT / AMM がデプロイされる

Step 4  エージェントの Approve を設定（初回1回のみ）
        npx ts-node scripts/setup-agent.ts

Step 5  .env にコントラクトアドレスを記入
        JPYY_ADDRESS=0x...
        YTT_ADDRESS=0x...
        AMM_ADDRESS=0x...
```

### デモ当日の手順

```
1  バックエンドサーバーを起動
   cd ai-agent && pnpm dev:be
   → ダミーモード（ブロックチェーン不要）で起動可能

2  フロントエンドを起動
   cd ai-agent && pnpm dev:fe
   → http://localhost:3000

3  ブラウザで http://localhost:3000/admin を開いて管理操作を行う
   └── JPYY を発行（エージェントウォレットに自動送信）
   └── AMMに初期流動性を追加

4  http://localhost:3000 を開いてエージェントウォレットを MetaMask で接続する
   └── 管理ウォレットと異なるアカウントで接続する（同じアドレスは拒否される）
   └── Polygon Amoy Testnet に切り替えが求められる場合は承認する

5  「▶ スタート」ボタンを押してエージェントを起動する
   └── AIが自動で売買を開始する

6  デモ参加者にブラウザの URL を共有して観覧してもらう
```

### デモ中の管理操作

| 状況 | 対応 |
|------|------|
| エージェントの JPYY が減った | 管理画面でエージェントに JPYY を追加配布 |
| YTT 価格を動かしたい | 管理画面でリバランス（setReserves）を実行 |
| POL が不足してきた | Faucet で補給 |
| AIの動作が激しすぎる | 慎重モードに切り替え、取引量を下げる |
| AIの動作が少なすぎる | 積極モードに切り替え、取引量を上げる |

## 9. システム構成

### 技術スタック

```
フロントエンド
  └── Next.js 14 + TypeScript（Pages Router）
  └── Zustand（walletStore・tradingStore）
  └── Recharts（価格チャート）
  └── ethers.js v6（MetaMask接続・POL残高取得）
  └── HTTP ポーリング（GET /api/state 5秒間隔・WebSocketなし）

バックエンド
  └── Node.js + TypeScript + Fastify
  └── @anthropic-ai/sdk（Claude API）
  └── ethers.js v6（ブロックチェーン操作）
  └── ws（WebSocketサーバー）

ブロックチェーン
  └── Polygon Amoy Testnet（chainId: 80002）
  └── Solidity 0.8.20
  └── OpenZeppelin（ERC-20・Ownable・AccessControl）
  └── Hardhat（開発・テスト・デプロイ）

AI
  └── Claude API（claude-sonnet-4-20250514）
```

### コントラクト構成

```
JPYY.sol   ERC-20 + AccessControl（管理者のみmint可能）
YTT.sol    ERC-20 + Ownable（管理者のみmint可能）
AMM.sol    定数積方式 DEX
           ├── swapJpyyForYtt()   BUY（エージェントが呼ぶ）
           ├── swapYttForJpyy()   SELL（エージェントが呼ぶ）
           ├── addLiquidity()     流動性追加（管理者のみ）
           └── setReserves()      価格変更（管理者のみ・デモ専用）
```

### ポーリングによる状態取得

フロントエンドは `GET /api/state` を **5 秒ごと** に呼び出して全画面を更新する。WebSocket は使用しない。

| エンドポイント | 方式 | 用途 |
|------------|------|------|
| `GET /api/state` | 5秒ポーリング | エージェント状態・残高・価格履歴・TX履歴を一括取得 |
| `POST /agent/connect` | ウォレット接続時に1回 | エージェントアドレスをバックエンドに登録 |
| `POST /agent/disconnect` | ウォレット切断時に1回 | エージェントアドレスをクリア |

### 環境変数

```env
# AIサービス
ANTHROPIC_API_KEY=sk-ant-...

# ネットワーク
RPC_URL=https://rpc-amoy.polygon.technology
CHAIN_ID=80002

# ウォレット（秘密鍵は .gitignore 済みの .env にのみ記載）
ADMIN_PRIVATE_KEY=0x...      # 管理者ウォレット（バックエンドが署名・MetaMask不要）
# エージェントウォレットは起動時にMetaMask接続で取得（.envに秘密鍵を書かない）

# コントラクト（デプロイ後に記入）
JPYY_ADDRESS=0x...
YTT_ADDRESS=0x...
AMM_ADDRESS=0x...
```
